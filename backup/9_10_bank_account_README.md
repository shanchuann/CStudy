# 模拟银行账户操作

## 问题描述
使用结构体和枚举实现一个简单的银行账户管理系统，支持不同类型的账户和交易操作。

## 要求
1. 定义账户类型枚举，包含：
   - 储蓄账户
   - 支票账户
   - 信用卡账户
2. 定义交易类型枚举，包含：
   - 存款
   - 取款
   - 转账
   - 利息
3. 定义交易记录结构体，包含：
   - 交易ID
   - 交易类型
   - 交易金额
   - 交易日期
   - 交易描述
4. 定义银行账户结构体，包含：
   - 账号
   - 账户名
   - 账户类型
   - 余额
   - 利率
   - 交易历史
   - 交易数量
5. 实现以下功能：
   - 创建新账户
   - 存款
   - 取款
   - 转账
   - 计算利息
   - 显示账户信息
   - 显示交易历史

## 输入输出格式
### 输入
- 创建账户时输入：
  - 账号（整数）
  - 账户名（字符串）
  - 账户类型（枚举值）
  - 初始余额（浮点数）
- 交易操作时输入：
  - 交易类型
  - 交易金额
  - 交易描述

### 输出
- 显示账户信息时，包含：
  - 账号
  - 账户名
  - 账户类型
  - 余额
  - 利率
  - 交易数量
- 显示交易历史时，包含：
  - 交易ID
  - 交易类型
  - 交易金额
  - 交易日期
  - 交易描述

## 示例
```
银行账户操作程序
==============

初始账户信息:
账号: 1001
账户名: 张三
账户类型: 储蓄账户
余额: 1000.00
利率: 2.00%
交易数量: 0
------------------------

交易后的账户信息:
账号: 1001
账户名: 张三
账户类型: 储蓄账户
余额: 1000.00
利率: 2.00%
交易数量: 4
------------------------

交易历史:
交易ID: 1
类型: 存款
金额: 500.00
日期: 2024-03-20
描述: 工资存入
------------------------
```

## 提示
1. 使用枚举类型定义账户和交易类型
2. 使用结构体组织账户和交易数据
3. 注意交易金额的正负处理
4. 考虑交易历史的大小限制
5. 可以使用数组存储交易历史
6. 注意字符串操作的安全性
