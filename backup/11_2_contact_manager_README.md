# 动态内存实现的通讯录管理系统

## 问题描述
实现一个使用动态内存管理的通讯录系统，支持添加、删除、查找和显示联系人信息。系统应该能够根据需求自动扩展内存空间。

## 要求
1. 实现以下功能：
   - 动态分配内存存储联系人信息
   - 自动扩展内存容量
   - 添加新联系人
   - 删除联系人
   - 查找联系人
   - 显示所有联系人
   - 正确释放内存

2. 系统特点：
   - 使用动态内存管理
   - 支持自动扩容
   - 高效的内存使用
   - 完整的错误处理
   - 用户友好的界面

## 数据结构
1. 联系人结构体：
   ```c
   typedef struct {
       char name[MAX_NAME_LEN];
       char phone[MAX_PHONE_LEN];
   } Contact;
   ```

2. 通讯录结构体：
   ```c
   typedef struct {
       Contact* contacts;
       int size;
       int capacity;
   } ContactBook;
   ```

## 主要功能
1. 内存管理：
   - 初始化通讯录
   - 扩展内存容量
   - 释放内存

2. 联系人操作：
   - 添加联系人
   - 删除联系人
   - 查找联系人
   - 显示所有联系人

## 输入输出格式
### 输入
- 通过菜单选择操作
- 输入联系人信息（姓名、电话）
- 输入要查找或删除的联系人姓名

### 输出
- 显示操作菜单
- 显示操作结果
- 显示联系人列表
- 显示错误信息

## 提示
1. 使用malloc和free进行内存管理
2. 使用realloc进行内存扩展
3. 注意内存泄漏问题
4. 添加适当的错误处理
5. 使用安全的字符串操作
