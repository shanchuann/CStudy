# 练习9.5：用联合体检测系统大小端

## 题目描述
编写一个程序，使用联合体（union）来检测系统的字节序（大小端）。要求：
- 使用联合体实现字节序检测
- 显示内存中的字节顺序
- 显示不同数据类型的字节序
- 提供详细的字节序信息

## 要求
1. 实现联合体结构
   - 定义整数和字符数组的联合体
   - 使用4字节整数作为测试值
   - 通过字符数组访问各个字节

2. 实现检测功能
   - 检测系统大小端
   - 显示内存中的字节顺序
   - 显示字节的物理地址
   - 解释每个字节的含义

3. 实现数据类型测试
   - 测试short类型（2字节）
   - 测试int类型（4字节）
   - 测试long类型（4字节）
   - 显示各类型的字节序

4. 实现信息显示
   - 显示测试值
   - 显示字节顺序
   - 显示内存地址
   - 提供字节含义说明

## 输入格式
- 程序无需输入
- 自动进行检测和显示

## 输出格式
- 显示系统字节序类型
- 显示内存中的字节顺序
- 显示字节的物理地址
- 显示不同数据类型的字节序

## 示例
输出：
```
系统字节序检测程序
==================

测试值：0x12345678
内存中的字节顺序：
字节 0: 0x78
字节 1: 0x56
字节 2: 0x34
字节 3: 0x12

系统是小端序（Little Endian）

字节序详细信息：
----------------------------------------
内存地址    字节值    含义
----------------------------------------
0x7fff1234  0x78    最低有效字节（LSB）
0x7fff1235  0x56    中间字节
0x7fff1236  0x34    中间字节
0x7fff1237  0x12    最高有效字节（MSB）

不同数据类型的字节序：
----------------------------------------
short (2字节): 0x34 0x12
int (4字节): 0x78 0x56 0x34 0x12
long (4字节): 0x78 0x56 0x34 0x12
```

## 提示
1. 使用联合体共享内存空间
2. 注意字节序的判断方法
3. 理解大小端的概念
4. 注意内存地址的显示
5. 考虑不同数据类型的字节序
